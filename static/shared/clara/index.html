Create, revise, or follow an existing style

Tell Clara what the presentation must achieve. You can start from source material or from a deck that already exists.

New presentation

Start from your materials

Add Word, PDF, PowerPoint, images, spreadsheets, notes, or recordings. State the audience, purpose, and duration; Clara proposes the structure and prepares the presentation.

"Create a 20-minute presentation for this audience from the materials in this folder."

Existing presentation

Describe what must change

Write your corrections or record them aloud. Clara works on a copy, changes what you requested, and checks that unrelated content remains unchanged.

"Record my feedback, apply it to a copy, and leave everything else unchanged."

Your visual style

Use the examples you already have

Add a template, a representative presentation, your logo, or existing brand guidance. Clara uses those references for layout and visual style.

"Use this PowerPoint as the visual reference for the new presentation."

Choose the final format

Choose PowerPoint when the file must remain editable. Choose a browser presentation when you want to present directly from Chrome.

PowerPoint PPTX

An editable PowerPoint

A normal .pptx file that can be opened and changed in PowerPoint. It is the right choice when you must follow an existing template or hand the file to someone else.

"Deliver an editable PowerPoint and check every slide."

Browser presentation

A presentation that opens in Chrome

A self-contained 16:9 presentation with speaker notes, fullscreen mode, keyboard or touch navigation, and optional animation. It does not require PowerPoint.

"Deliver a browser presentation with speaker notes."

When the project is more than a presentation

If the work includes many documents, conversations, and versions, Clara keeps the project together in its folder so that you can continue it in a later Codex session.

The sources stay identifiable

Tell Clara which documents are authoritative and which files are only visual or contextual references.

Open questions stay visible

Clara can keep track of missing information, points to confirm, and corrections that must be reflected in the next draft.

The work continues across sessions

Close Codex, reopen the same folder in another session, and continue from the existing materials and drafts.

From product data to a checked retail report

Retailer Signals and Brand Fit are available now. The server keeps structured product data, the central taxonomy, accepted mappings, and image URLs; model work runs in Codex without a user or server API key. User product images and reports remain local unless deliberately shared.

01

Collect product data. Retailer pages and images are captured locally; stored product records and image URLs feed the central product dataset. Fresh local scrape ingestion is not yet part of installed Clara.

02

Map attributes. Lower-cost Codex agents map unresolved cells to the central taxonomy; independent agents review them and the server validates accepted mappings.

03

Compare cohorts. New means labelled new by the retailer; new products and best sellers are compared with the remaining products using checked calculations.

04

Create and check the report. Clara writes a private HTML report locally and checks its calculations, evidence, rendering, and readability.

Brand Fit

Start from a completed, checked Retailer Signals analysis. Clara compares those signals with the brand's current presence at the selected retailer and the brand-owned catalogue in the stored database snapshot—not a live shelf check. Codex interprets the fit locally; product images and the HTML report stay on your computer, and neither the user nor the server needs a model API key.

"Compare these retailer signals with this brand's current presence at the retailer and its owned catalogue."

Other Clara workflows

Clara also handles interviews, recordings, retail attributes, and business-data charts as distinct workflows.

Remote interview

Interview someone with a personal link

The participant opens the link without signing in, authorizes the microphone, and speaks with an AI interviewer for up to 15 minutes. Questions adapt to the answers; the participant's spoken responses are recorded and transcribed.

"Prepare a 15-minute interview in Italian for this participant."

Transcription

Transcribe a meeting or debrief

Record the conversation or add an existing audio file. Clara prepares a transcript, helps identify the speakers, and adds the reviewed material to the project.

"Transcribe this recording and identify the speakers."

Retail analysis

Map attributes and compare assortments

Codex maps products to the central taxonomy, compares new products or best sellers with the rest, and checks the private HTML report.

It currently uses stored product data; product images and the report stay on your computer.

"Compare this retailer's new products with the rest and tell me whether the report is correct."

Business data

Analyze a dataset and make useful charts

Codex chooses the analysis from the business question and actual fields; Clara runs checked calculations and rendering for distributions, funnels, mix, period comparisons, scatter, overlap, statements, and variance.

"Analyze this spreadsheet, choose the most useful chart, and explain the result."

Install Clara in Codex

Install the published Clara release from its official OpenAI Marketplace page.

  • Open Clara's official OpenAI Marketplace page.
  • Use the install action shown on the listing.
  • Open Codex, go to Plugins, and confirm that Clara is installed and enabled.
  • Open a Codex chat in the project folder and ask Clara to start there.
If installation does not work
Install action unavailable Confirm that you are signed into OpenAI, then reopen Clara's official Marketplace page.
Clara does not appear Confirm that Clara is installed and enabled, then open a new Codex chat in the project folder.

"Use Clara in this folder."
