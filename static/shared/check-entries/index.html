Support testing

Test sampled entries against support PDFs.

Compare selected journal rows with invoices, transfers, vouchers and other readable support.

Give it the sampled entries and a folder of support PDFs. Get back row-level statuses, mismatches, missing support and manual-review cases.

Give it / get back
  • Give it sampled entries plus a folder of support PDFs.
  • Confirm fields, amount tolerance and date window.
  • Get back ok, mismatch, missing_support and manual_review rows.

What it does

From selected entries to support checks

Use it when sample rows are known but support is scattered across local PDFs. Codex asks only for field and tolerance choices that affect the check.

Inputs

What it reads

Works with local files already available to the firm. OCR-only scanned PDFs are flagged for review rather than treated as checked.

Outputs

What it produces

The result is a reviewable work folder: recipe, normalized entries, PDF inventory, result table, reading checks and review trail.

    Entry checks tied to documents

    The inspection step proposes movement number, date, amount, debit/credit and beneficiary fields, then asks only for unresolved choices.

    The result separates normalized entries, PDF inventory, comparison outcomes and review settings so mismatches can be reviewed row by row.

    Unreadable PDFs and weak matches are labeled for manual review instead of being treated as passed.

    Use

    Create support checks from one folder

    Open the folder, confirm the entry fields and tolerances, then generate the check results, reading checks and review trail.
