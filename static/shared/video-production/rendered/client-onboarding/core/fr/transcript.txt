Vera | Le modèle d’information de l’onboarding client

Cette édition Core explique la structure commune de l’onboarding client. Le modèle relie cinq éléments : les preuves sources et leur provenance ; l’identité du client et les personnes impliquées ; les services demandés et les faits de la mission ; les demandes ouvertes avec les actions suivantes ; enfin les dates et l’historique conservés dans le temps. Les éléments restent distincts mais connectés, afin de rattacher chaque information au document ou à la décision dont elle provient. Les implémentations de chaque pays rendent le modèle opérationnel en associant leurs documents, contrôles et règles à ces champs stables. Le résultat conceptuel est une structure lisible pour comprendre ce qui est connu, ce qui reste ouvert et comment le dossier évolue.
