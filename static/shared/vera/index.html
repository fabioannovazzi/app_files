Dati · confini reali

Vera lavora sui dati reali del cliente.

Vera e Clara non anonimizzano automaticamente i dati. Possono usare Python in locale per filtrare o aggregare le informazioni quando è utile.

I dati forniti al modello vengono trattati attraverso il piano ChatGPT già utilizzato dall’utente. I workflow ordinari non inviano a Mparanza file dei clienti, prompt o contenuti del contesto del modello.

Dati del cliente

Se identità, titolarità, fatti fiscali, contabili o previdenziali servono alla revisione, possono entrare nel contesto di Codex.

Elaborazione locale

Python in locale può filtrare, aggregare, calcolare e organizzare quando aiuta il lavoro. Non viene usato per cancellare identificativi a prescindere.

Scelta dell’account

Per le normali funzioni dentro Codex, lo studio usa il piano ChatGPT esistente e configura i controlli disponibili per quel piano.

Credenziali e sessioni

Password, chiavi API, cookie, token e dati di sessione non devono entrare nei prompt o nei file leggibili da Codex.

Due categorie di trattamento.

Normali funzioni dentro Codex

Ogni workflow viene mappato quando viene aggiunto o quando cambia: che cosa resta normalmente locale e che cosa può leggere Codex. Non si crea una registrazione per ogni prompt.

Servizi hosted da Mparanza

Un servizio hosted è un confine separato e viene documentato una volta a livello di servizio. Ricerche, connector, portali o invii scelti dall’utente seguono invece le condizioni del servizio esterno; non sono una terza categoria Mparanza.

La stessa regola vale per Vera e Clara: una mappa per workflow, una descrizione per servizio hosted e nessuna documentazione prompt per prompt.
