PNG Examples Gallery
====================

`runs/png_examples/png-gallery/index.html` is the curated visual check for the
chart examples generated by the plugin scripts. A local static copy can be
built at `static/shared/png-gallery/index.html`; generated galleries are not
committed.

The gallery is not only a visual QA page. It is also the chart/table inventory
used to prove that reporting consumers can discover each visual, understand
when to invoke it, pass the required parameters, and receive both an image and
model-readable data.

For the selection-side contract built on top of gallery evidence, see
`docs/chart_selection_manifest.md`.

Every gallery card must therefore be artifact-ready:

- no broken links;
- a real source artifact (`.html` or `.png`);
- context and manifest/provenance sidecars;
- model-readable data, preferably chart-level CSV and otherwise a canonical or
  summary table returned by the run;
- a recipe or equivalent invocation contract;
- an `artifact_contract` with capability id, when-to-use guidance, required
  parameters, optional parameters, source reference fields, and expected outputs.

HTML-backed source artifacts are allowed. They are the repeatable source for
screenshot capture when direct PNG export is not appropriate. The required
gallery output is still a PNG preview image.

Operational rule
----------------

Run chart examples from the project virtual environment and make sure the local
runtime can launch the browser/export dependencies used by the plugins:

```bash
source .venv/bin/activate
```

For Plotly-backed examples, this means Kaleido must be usable by the same
environment. For screenshot fallbacks, Chrome must be launchable by the local
runtime. If those processes are blocked by the sandbox or local permissions,
the generator may leave HTML-only artifacts even though direct PNG export would
work in a normal run.

The local run gallery build is:

```bash
python scripts/build_png_examples_gallery.py
python scripts/validate_png_gallery_manifest.py \
  --allow-html \
  --require-artifact-ready
```

The local static gallery preserves the existing PNG previews and republishes
source/context/data/manifest/recipe sidecars beside them:

```bash
python scripts/build_static_png_gallery_inventory.py
python scripts/validate_png_gallery_manifest.py \
  --manifest static/shared/png-gallery/manifest.json \
  --allow-html \
  --require-artifact-ready
```

`validate_png_gallery_manifest.py` checks only the selected gallery manifest. It
intentionally does not fail on all HTML files under `runs/`, because excluded
examples, stale debug runs, and sidecar HTML files can remain useful for
diagnosis.

Reporting visual review
-----------------------

Use `scripts/prepare_reporting_visual_review.py` when a gallery chart or table
needs reporting visual review before editing. It reads the generated gallery
manifest, resolves the exact PNG/source/context sidecars, infers the reporting
visual family and variants, and attaches classified reference examples from
`docs/visual_reporting_references.json`.

For IBCS-style reporting chart reviews, treat a single visible font size across
the whole chart as a required check. Titles, axis labels, tick labels, value
labels, legends, captions, and category labels should use the same size; create
hierarchy with position, spacing, weight, and marks instead of mixed font sizes.

For one artifact:

```bash
python scripts/prepare_reporting_visual_review.py --only "column_total"
```

For an all-gallery review queue:

```bash
python scripts/prepare_reporting_visual_review.py \
  --format json \
  --output runs/reporting_visual_review_queue.json
```

When the validator fails
------------------------

If the validator reports HTML-backed gallery cards without `--allow-html`:

1. Decide whether direct PNG source export is actually required for that
   workflow. For artifact readiness, HTML source plus PNG preview is valid.
2. If direct PNG source is required, rerun the owning plugin example generator
   from `.venv` with Chrome/Kaleido access.
3. Rebuild the gallery with `python scripts/build_png_examples_gallery.py`.
4. Inspect `python scripts/audit_png_example_artifacts.py` if the card is still
   HTML-backed; the audit output can show renderer labels and export errors.

If `--require-artifact-ready` reports missing source, context, data,
manifest, recipe, or contract fields, fix the owning plugin/run artifact or the
static inventory publisher. Do not leave visual-only cards in the gallery.
